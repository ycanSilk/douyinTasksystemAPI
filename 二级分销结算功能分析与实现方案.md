# 二级分销结算功能分析与实现方案

## 1. 现有结算功能分析

### 1.1 任务完成结算流程
- **触发条件**：任务审核通过
- **结算逻辑**：
  - 发放佣金给C端用户
  - 检查C端用户是否有上级代理团长，如有则发放一级团长佣金
  - 佣金计算基于任务模板配置
- **实现文件**：`api/b/v1/tasks/review.php`

### 1.2 租赁订单结算流程
- **触发条件**：租赁订单到期
- **结算逻辑**：
  - 发放收益给卖方
  - 检查订单是否关联了代理团长，如有则发放一级团长佣金
  - 佣金在订单创建时计算并存储
- **实现文件**：`cron/rental_check.php`、`api/rental/orders/create-from-offer.php`

### 1.3 现有系统限制
- 只支持一级代理佣金发放
- 未实现二级代理佣金结算
- 代理层级查询只支持一级

## 2. 二级分销结算功能设计

### 2.1 核心修改点
1. **用户层级查询**：实现最多两级上级团长信息查询
2. **佣金计算**：支持二级团长佣金的独立计算
3. **佣金发放**：为两级团长分别发放对应金额的佣金
4. **事务处理**：确保佣金发放的事务完整性
5. **日志记录**：增加详细的结算日志

### 2.2 技术实现方案

#### 2.2.1 任务完成结算修改（api/b/v1/tasks/review.php）
1. **修改佣金计算逻辑**：
   - 从任务模板中读取二级代理佣金配置
   - 计算一级和二级代理佣金

2. **修改层级查询逻辑**：
   - 查询当前用户的直接上级团长
   - 再查询上级团长的上级团长（二级上级）
   - 检查两级上级是否为代理

3. **修改佣金发放逻辑**：
   - 为一级代理发放佣金
   - 为二级代理发放佣金
   - 记录详细的钱包流水

#### 2.2.2 租赁订单结算修改
1. **订单创建时修改**（api/rental/orders/create-from-offer.php）：
   - 计算一级和二级代理佣金
   - 存储两级代理信息和佣金金额

2. **订单结算时修改**（cron/rental_check.php）：
   - 为一级代理发放佣金
   - 为二级代理发放佣金

## 4. 数据库结构修改

### 4.1 任务模板表修改
根据分析数据库结构，`task_templates` 表已经包含了一级代理佣金的配置字段：
- `agent_commission`：普通团长佣金
- `senior_agent_commission`：高级团长佣金
- `stage1_agent_commission`：组合任务阶段1普通团长佣金
- `stage1_senior_agent_commission`：组合任务阶段1高级团长佣金
- `stage2_agent_commission`：组合任务阶段2普通团长佣金
- `stage2_senior_agent_commission`：组合任务阶段2高级团长佣金

由于二级分销结算的奖励金额与现有一级代理团长的结算奖励金额一致，因此**不需要修改 `task_templates` 表结构**，可以直接复用现有的佣金字段。

需要在 `rental_orders` 表中添加二级代理相关字段：
- `second_agent_user_id`：二级代理用户ID
- `second_agent_amount`：二级代理佣金金额

### 4.3 配置表修改
根据分析数据库结构，`app_config` 表已经包含了一级代理分成比例的配置：
- `rental_agent_rate`：普通代理分成比例
- `rental_senior_agent_rate`：高级代理分成比例

由于二级分销结算的奖励金额与现有一级代理团长的结算奖励金额一致，因此**不需要修改 `app_config` 表结构**，可以直接复用现有的配置项。

通过以上修改，系统将实现二级分销结算模式，为完成任务和租赁订单的用户的两级上级代理团长自动发放佣金。这将激励代理团长发展更多的下级用户，促进平台的快速增长。同时，系统通过完善的事务处理和日志记录，确保佣金结算的准确性和可靠性。
