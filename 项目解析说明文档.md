# TaskSystem 项目解析说明文档

## 1. 项目概览

TaskSystem 是一个基于 PHP 原生开发的任务平台后端系统，主要提供任务发布、接受、提交、审核等功能，同时包含用户管理、钱包管理、代理系统和租赁系统等模块。

### 1.1 核心功能

- **用户管理**：支持 B 端（任务发布方）和 C 端（任务执行方）用户注册、登录、密码管理
- **任务管理**：任务发布、接受、提交、审核、超时处理
- **钱包系统**：充值、提现、钱包密码管理、交易记录
- **代理系统**：普通代理和高级代理申请、管理、收益统计
- **租赁系统**：设备租赁需求发布、报价、订单管理、售后工单
- **通知系统**：系统通知、消息管理

## 2. 项目架构

### 2.1 目录结构

```
tasksystem/
├── api/             # API 接口目录
│   ├── b/           # B 端接口（任务发布方）
│   ├── c/           # C 端接口（任务执行方）
│   └── rental/      # 租赁系统接口
├── config/          # 配置文件目录
├── core/            # 核心类目录
├── cron/            # 定时任务目录
├── img/             # 图片存储目录
└── task_admin/      # 后台管理系统
```

### 2.2 分层架构

- **API 层**：处理 HTTP 请求，参数校验，调用核心类处理业务逻辑，返回响应
- **核心层**：提供通用功能，如数据库连接、认证、响应处理等
- **数据层**：使用 MySQL 数据库存储数据

### 2.3 核心模块

| 模块 | 职责 | 文件位置 |
|------|------|----------|
| 数据库连接 | 管理数据库连接（单例模式） | core/Database.php |
| 认证管理 | Token 生成、校验、跨端防护 | core/Token.php, core/AuthMiddleware.php |
| 响应处理 | 统一 JSON 响应格式 | core/Response.php |
| 配置管理 | 从数据库读取配置，带缓存机制 | core/AppConfig.php |
| 通知系统 | 处理系统通知 | core/Notification.php |
| 钱包密码 | 钱包密码加密、验证 | core/WalletPassword.php |
| 邀请码 | 生成和验证邀请码 | core/InviteCode.php |

## 3. 技术实现

### 3.1 认证机制

- **Token 生成**：使用 base64(JSON) 格式，包含用户 ID、用户类型和过期时间
- **Token 校验**：验证 token 格式、过期时间、用户类型匹配、数据库一致性
- **跨端防护**：防止 B 端用户访问 C 端接口，反之亦然
- **认证中间件**：自动识别 API 端，进行 token 校验

### 3.2 数据库设计

- **用户表**：b_users（B 端用户）、c_users（C 端用户）
- **任务表**：b_tasks（任务发布）、c_task_records（任务执行记录）
- **钱包表**：wallets（钱包）、wallet_logs（钱包日志）
- **代理表**：agents（代理）、agent_applications（代理申请）
- **租赁表**：rental_demands（租赁需求）、rental_offers（租赁报价）、rental_orders（租赁订单）、rental_tickets（租赁工单）
- **配置表**：app_config（系统配置）

### 3.3 接口设计

- **RESTful 风格**：使用 HTTP 方法（GET、POST、PUT、DELETE）区分操作
- **统一响应格式**：
  ```json
  {
    "code": 0,           // 0 表示成功，非 0 表示错误
    "message": "ok",    // 响应消息
    "data": {},          // 响应数据
    "timestamp": 1234567890  // 时间戳
  }
  ```
- **认证方式**：支持两种 Token 传递方式
  - Authorization: Bearer <token>
  - X-Token: <token>

### 3.4 安全措施

- **密码加密**：使用 password_hash() 函数加密存储密码
- **输入校验**：对所有用户输入进行严格校验
- **Token 过期**：Token 默认有效期 7 天
- **SQL 注入防护**：使用 PDO 预处理语句
- **跨端访问防护**：防止不同端用户互相访问接口

## 4. 核心功能模块

### 4.1 用户认证模块

- **注册**：支持邮箱、手机号注册
- **登录**：支持用户名、邮箱、手机号登录
- **密码管理**：修改密码、忘记密码
- **Token 管理**：刷新 Token、退出登录

### 4.2 任务管理模块

- **任务发布**：B 端用户发布任务，设置任务类型、奖励金额、任务数量等
- **任务接受**：C 端用户接受任务
- **任务提交**：C 端用户提交任务完成情况，包括截图、评论等
- **任务审核**：B 端用户审核任务完成情况，通过或驳回
- **任务超时**：系统自动处理超时任务

### 4.3 钱包管理模块

- **充值**：B 端用户充值到钱包
- **提现**：C 端用户从钱包提现
- **钱包密码**：设置、修改、验证钱包密码
- **交易记录**：查看钱包交易历史

### 4.4 代理系统模块

- **代理申请**：C 端用户申请成为代理
- **代理管理**：查看代理信息、统计数据
- **高级代理**：满足条件的代理可申请成为高级代理
- **代理收益**：代理可获得下线用户的任务收益分成

### 4.5 租赁系统模块

- **需求发布**：发布设备租赁需求
- **报价管理**：对需求进行报价
- **订单管理**：创建订单、查看订单状态、订单续期
- **工单系统**：处理租赁过程中的问题

### 4.6 通知系统模块

- **通知列表**：查看系统通知
- **通知详情**：查看通知详细内容
- **标记已读**：标记通知为已读

## 5. 定时任务

- **任务检查**：检查任务超时情况
- **租赁检查**：检查租赁订单状态

## 6. 配置管理

- **动态配置**：从数据库读取配置，支持实时更新
- **配置缓存**：配置缓存 5 分钟，提高性能
- **默认配置**：数据库不可用时使用默认配置

## 7. 系统特点

1. **多端分离**：清晰的 B 端和 C 端分离设计
2. **模块化架构**：各功能模块独立，便于维护和扩展
3. **安全可靠**：多重安全措施，保障系统安全
4. **灵活配置**：支持动态配置，适应不同场景需求
5. **高性能**：缓存机制、数据库优化，提高系统响应速度
6. **易扩展**：模块化设计，便于添加新功能

## 8. 技术栈

- **后端语言**：PHP 7+
- **数据库**：MySQL
- **认证**：自定义 Token 系统
- **HTTP 服务**：Apache/Nginx
- **数据缓存**：内存缓存（配置缓存）

## 9. 部署说明

1. **环境要求**：
   - PHP 7.0 或更高版本
   - MySQL 5.6 或更高版本
   - Apache 或 Nginx 服务器

2. **部署步骤**：
   - 克隆代码到服务器
   - 创建数据库并导入数据结构
   - 配置数据库连接信息（config/database.php）
   - 配置 Web 服务器，设置根目录为 task system
   - 确保 img 目录可写
   - 设置定时任务（cron/task_check.php 和 cron/rental_check.php）

3. **访问地址**：
   - API 接口：http://your-domain/api/{b|c|rental}/v1/
   - 后台管理：http://your-domain/task_admin/

## 10. 总结

TaskSystem 是一个功能完整、架构清晰的任务平台后端系统，采用 PHP 原生开发，具有良好的扩展性和安全性。系统支持任务管理、用户管理、钱包管理、代理系统和租赁系统等核心功能，满足了任务平台的各种业务需求。

系统的设计理念是模块化、安全可靠、高性能，通过分层架构和核心类的设计，使得代码结构清晰，易于维护和扩展。同时，系统提供了丰富的 API 接口，支持前端应用的开发和集成。

作为一个后端系统，TaskSystem 为任务平台的正常运行提供了稳定的技术支持，是一个值得学习和参考的 PHP 项目。
